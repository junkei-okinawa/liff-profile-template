# 05. オペレーションガイド

本ドキュメントは、アプリケーションの運用担当者（ビジネス担当者）に向けたガイドである。
開発知識がなくても可能な、コンテンツの更新方法やユーザー管理について説明する。

## 1. 利用規約の更新方法

利用規約の文言を変更したい場合は、以下のファイルを編集する。

- **対象ファイル**: `public/terms.md`

このファイルは「Markdown（マークダウン）」という形式で書かれている。
メモ帳などのテキストエディタで開き、文字を修正して保存・デプロイ（開発担当者に依頼）することで、アプリ上の表示が更新される。

### 記述例
```markdown
# 利用規約

## 第1条（目的）
本規約は...

## 第2条（定義）
...
```

## 2. 退会確認メッセージの更新方法

退会画面に表示される注意書きを変更したい場合は、以下のファイルを編集する。

- **対象ファイル**: `public/unsubscribe.md`

同様にMarkdown形式で記述する。

## 3. ユーザーの同意状況の確認

ユーザー情報は機密情報にあたるため、別途用意される**管理画面**にて確認を行う予定である。
Google Cloud Console等のデータベース管理画面への直接アクセスは、セキュリティおよび運用ミスの防止の観点から推奨されない。
管理画面が提供されるまでは、開発担当者に確認を依頼すること。

## 4. お問い合わせ対応

ユーザーから「退会したのにログインできてしまう」等の問い合わせがあった場合：
- 現在の退会機能は、詳細仕様が未確定のため、**暫定的に「ログアウトのみ」を行う仕様**となっている。
- 今後、仕様が確定次第、正式な退会処理（データ削除や連携解除等）が実装される予定である。
- ユーザーには「現在はログアウトのみ行われます。正式な退会機能は後日実装予定です」と案内すること。
